# EPQ 工作台文档分析、个性化设置与日期修复设计

## 目标

在现有 EPQ Camp Companion 上完成四类改进：

1. 美化登录页面，并将工作台默认视觉改为专业蓝橙配色。
2. 修复学生营地开始日期与每日记录日期、营地天数脱节的问题。
3. 支持上传并长期保存学生文档和提取文本，由 DeepSeek 根据当日材料生成 AO1–AO4 观察建议。
4. 增加跟随账号同步的外观与 DeepSeek 设置，每位助教使用自己的 API Key。

## 已确认的产品决策

- 视觉方向采用 A「专业营地」。
- 文档原文件及提取文本长期保存。
- 支持 PDF、Word、TXT、Excel、PowerPoint、JPG 和 PNG。
- 单文件上限为 25 MB。
- AI 生成 AO1–AO4 全部建议，先预览、编辑和勾选，再由助教采用。
- 设置跟随账号跨设备同步。
- 主题提供预设和自定义颜色。
- 每位助教必须配置自己的 DeepSeek API Key。
- 营地开始日为第 1 天，开始日前禁止创建每日记录。
- 文档属于某个学生和营地日，上传时允许选择营地第几天。
- AO 分析只参考所选营地日的文档和当日记录内容。
- 技术路径采用 Supabase 私有存储和可重试的异步解析任务。

## 视觉与界面

### 登录页

桌面端使用左右双栏布局：

- 左侧为深海蓝品牌区，说明每日记录、文档分析、AO 观察和连续反馈的价值。
- 右侧为白色登录/注册卡片。
- 橙色只用于主提交按钮、重点标签和状态强调。
- 保留现有邮箱验证、错误信息、加载动画和移动端无溢出要求。

移动端改为单列：品牌介绍缩短后位于表单上方，避免为视觉效果牺牲表单可用性。

### 工作台

默认主题为：

- 深海蓝：主导航、标题、重点文本。
- 中蓝：次级按钮、选中状态、信息卡片。
- 活力橙：新增、生成、确认等关键操作。
- 浅蓝灰：页面背景和非关键卡片。

设置入口位于工作台页眉右侧，与退出登录按钮分离。在窄屏下页眉自动换行。

### 设置面板

设置以抽屉或独立面板呈现，包含：

- 主题预设：专业蓝橙、森林绿、紫金、灰蓝。
- 自定义主色和强调色。
- 字体大小：小、标准、大。
- 字体粗细：常规、中等、加粗。
- DeepSeek API Key 状态。
- 模型预设、实际模型 ID 和连接测试。

自定义颜色保存前执行对比度检查。正文与背景、按钮文字与按钮背景未达到 WCAG AA 常规文本对比度时禁止保存，并指出需要调整的颜色。

设置从服务端随工作区初始数据加载。服务端在 HTML 根节点输出对应 CSS 变量和字体属性，避免客户端加载后主题闪烁。

## 营地日期与营地天数

### 当前问题根因

`students.camp_start_date` 当前只被保存和显示。每日记录表单的 `campDay` 空值始终硬编码为 `1`，工作区默认日期则独立采用上海时区的今天，因此开始日期没有参与每日记录日期和营地日计算。

### 新规则

营地天数统一由以下规则计算：

```text
campDay = calendarDays(recordDate - campStartDate) + 1
```

- 开始日期为第 1 天。
- 使用纯日历日期运算，避免时区和夏令时导致偏差。
- 开始日期之前的结果无效，不允许创建或保存记录。
- 表单中的营地天数改为只读显示，不再允许手动修改。
- API 和数据库保存函数必须重新计算并验证营地天数，不能信任浏览器提交值。

### 默认日期

- 今天早于开始日期时，工作区显示开始日期，但每日记录区锁定并提示“营地尚未开始”。
- 今天等于或晚于开始日期时，默认显示今天。
- 用户仍可切换到其他日期；开始日前的日期只读且不能保存。

### 修改开始日期

如果学生已有每日记录或文档，修改开始日期前检查：

- 是否会使已有记录落在开始日前。
- 是否会改变已有文档所标记的营地日。

存在冲突时不保存新日期，返回明确的冲突摘要。首版不自动批量迁移已有记录和文档，避免静默改变历史含义。

## 文档存储与数据模型

### 存储

创建私有 Supabase Storage 存储桶 `student-documents`。对象路径包含所有者和文档标识：

```text
{owner_id}/{student_id}/{document_id}/{sanitized_filename}
```

浏览器通过受限签名上传地址直接上传文件，避免 25 MB 文件经过 Vercel 函数。读取和下载使用短时签名 URL。

### 文档表

新增 `student_documents`，至少包含：

- `id`
- `owner_id`
- `student_id`
- `camp_day`
- `record_date`
- `original_filename`
- `storage_path`
- `mime_type`
- `size_bytes`
- `sha256`
- `parse_status`
- `parse_error_code`
- `extracted_text`
- `extracted_text_sha256`
- `page_or_sheet_count`
- `created_at`
- `updated_at`

`parse_status` 为：

- `uploaded`
- `queued`
- `parsing`
- `ready`
- `failed`

文档使用上传时的 `camp_day` 和按当时学生开始日期换算出的 `record_date`。修改学生开始日期不会静默重写历史文档。

### 任务表

新增 `document_jobs`：

- `id`
- `owner_id`
- `document_id`
- `kind`：`parse` 或 `analyze_ao`
- `status`：`queued`、`running`、`succeeded`、`failed`
- `attempt_count`
- `available_at`
- `locked_at`
- `error_code`
- `created_at`
- `updated_at`

任务以幂等方式执行。相同文档和文本摘要的成功任务不会重复消耗 OCR 或模型调用。

### AO 分析表

新增 `document_ao_analyses`，保存：

- 文档、学生、所有者和记录日期。
- 所用模型 ID。
- 分析输入摘要。
- AO1–AO4 的建议备注、引用证据和置信提示。
- 创建时间。

分析结果作为历史建议保存，不直接写入每日记录。

## 文件支持与提取

允许的文件：

- PDF
- DOC、DOCX
- TXT
- XLS、XLSX
- PPT、PPTX
- JPG、JPEG、PNG

同时检查扩展名、MIME 类型和文件签名字节，拒绝伪装类型。文件大小不得超过 25 MB。

解析策略：

- TXT：安全解码并限制最大提取字符数。
- DOCX、XLSX、PPTX：解析 OOXML 容器中的正文。
- 旧版 DOC、XLS、PPT：交由隔离的文档转换/提取执行器。
- 文本型 PDF：直接提取文本。
- 扫描 PDF、JPG、PNG：通过 OCR 提取文字。

由于 Vercel 免费运行时不适合长期运行 Office 转换和 OCR，解析器通过独立的可重试任务执行器调用。任务执行器只获得单个文件的短时签名下载 URL，并将提取结果返回受鉴权的任务回调。前端和数据库不依赖具体解析供应商，后续可以替换为自托管 worker 或托管文档解析/OCR 服务。

提取文本设置总字符上限并记录截断状态。原文件仍然保留，可重新解析。

## AO 分析

### 分析输入

用户选择一个已经解析完成的营地日后，系统将以下内容发送给 DeepSeek：

- 该学生该营地日所有选中文档的提取文本。
- 当日记录中的成果、证据、困难、明日计划和助教过程观察。
- 内置 AQA EPQ AO1 Manage、AO2 Use resources、AO3 Develop and realise、AO4 Review 的谨慎观察准则。

不会自动混入其他日期的每日记录、历史文档或历史反馈。

### 分析输出

每个 AO 输出：

- `suggestedNote`：适合助教记录的观察建议。
- `evidenceQuotes`：来自文档或当日记录的简短证据。
- `confidence`：`high`、`medium` 或 `low`。
- `caution`：证据不足或不能由文档判断时的说明。

模型必须区分“学生文档中出现的内容”和“助教观察到的行为”。无法从材料支持的 AO 应输出低置信度和谨慎提示，不能编造学生表现。

### 采用流程

分析结果在独立预览区显示。助教可以：

- 编辑建议。
- 单独勾选 AO1–AO4。
- 采用选中建议。

采用时：

- 默认不覆盖非空 AO 备注。
- 如果目标备注已有内容，显示逐项覆盖确认。
- 采用后仍由现有每日记录自动保存机制负责持久化。

## DeepSeek 设置与密钥安全

### 用户设置表

新增 `user_settings`，包含：

- `owner_id`
- `theme_preset`
- `primary_color`
- `accent_color`
- `font_scale`
- `font_weight`
- `deepseek_model_preset`
- `deepseek_model_id`
- `deepseek_key_ciphertext`
- `deepseek_key_iv`
- `deepseek_key_tag`
- `deepseek_key_last4`
- `deepseek_key_updated_at`
- `created_at`
- `updated_at`

### 密钥处理

- API Key 仅通过 HTTPS 发送给受鉴权的服务端设置接口。
- 使用部署环境中的独立 `SETTINGS_ENCRYPTION_KEY` 通过 AES-256-GCM 加密。
- 数据库只保存密文、随机 IV、认证标签和末尾四位。
- API 永不返回完整密钥。
- 日志、错误和分析记录不得包含密钥。
- 用户可以测试、替换或删除密钥。
- 删除后所有 DeepSeek 功能停用，直到重新配置。
- 服务端调用 DeepSeek 时按当前用户读取和解密密钥，不再回退到共享部署 Key。

### 模型

预设为：

- Chat：`deepseek-chat`
- Reason：`deepseek-reasoner`
- V4 Pro：用户可配置实际模型 ID
- 自定义：用户直接填写模型 ID

“V4 Pro”是产品预设名称，不硬编码未经验证的供应商模型 ID。保存 V4 Pro 或自定义配置前必须使用用户 API Key 执行轻量连接测试；接口不接受该模型 ID 时不保存。

## API 边界

新增服务端接口：

- 获取和更新用户外观设置。
- 保存、测试、替换和删除 DeepSeek API Key。
- 创建文件上传会话。
- 完成上传并创建解析任务。
- 查询学生文档和任务状态。
- 创建短时下载 URL。
- 重试解析。
- 删除文档及其文本和分析。
- 发起 AO 分析。
- 查询 AO 分析结果。

所有接口先验证当前 Supabase 用户，再按 `owner_id`、`student_id` 和文档关联关系校验所有权。

## 错误与状态

文档区域显示：

- 上传中
- 等待解析
- 正在解析
- 可以分析
- 正在分析
- 分析完成
- 失败，可重试

错误只返回稳定错误码和用户可理解的安全文案，不暴露供应商响应、存储路径、堆栈或密钥。

删除文档属于不可逆操作，需要二次确认。删除数据库行和存储对象时，优先标记删除并由任务清理，避免部分失败后出现孤立数据。

## 测试与验收

### 日期

- 开始日为第 1 天。
- 跨月、跨年正确。
- 开始日前只读且 API 拒绝保存。
- 浏览器与服务端计算一致。
- 修改开始日期时能检测已有数据冲突。

### 文件

- 所有支持格式可进入正确解析路径。
- 超过 25 MB、伪装类型、损坏文件被拒绝。
- 扫描 PDF 和图片进入 OCR。
- 文档严格隔离到当前用户、学生和营地日。
- 删除同时清理原文件、提取文本和分析。

### AO 分析

- 输入只包含所选营地日材料。
- 输出满足严格结构。
- 低证据场景产生谨慎提示。
- 建议不会自动覆盖人工备注。
- 采用选中建议后进入现有自动保存流程。

### 设置与安全

- 主题和字体跨设备同步。
- 自定义颜色对比度不合格时无法保存。
- 密钥加密、遮罩、替换、删除和账号隔离正确。
- 完整密钥不出现在 HTML、客户端状态、日志或 API 响应。
- 无有效密钥或模型测试失败时 AI 功能安全禁用。

### 响应式与流程

- 320px 手机、平板和桌面无控件重叠及横向溢出。
- 登录和注册保留邮箱验证。
- 覆盖登录、学生创建、日期计算、上传、解析、分析、采用 AO 建议和生成反馈的端到端流程。

## 分阶段交付

实现按可独立验证的顺序进行：

1. 日期计算根因修复。
2. 主题系统、登录页和设置外观。
3. 用户 DeepSeek 配置和加密密钥。
4. 私有文档存储与上传。
5. 异步文档解析和 OCR。
6. AO 分析、预览和采用。
7. 完整响应式、权限和生产部署验证。
