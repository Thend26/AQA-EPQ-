# 注册、响应式布局与加载状态设计规格

## 1. 目标

在现有 EPQ Camp Companion 生产网站上补齐公开邮箱注册，并修复窄屏设备上按钮、图标和日期控件重叠的问题。同时统一异步操作的视觉反馈，让用户能明确判断网站正在登录、注册、保存或调用 AI，而不是误以为点击无效。

本次改动保持现有 Next.js、Supabase、Vercel 和 DeepSeek 架构，不引入管理员审批、邀请码、第三方登录或新的角色权限。

## 2. 已确认问题与根因

### 2.1 无法创建账号

当前登录组件只接收 `signIn`，登录页只调用 `signInWithPassword`，现有组件测试和端到端测试还明确断言页面不显示注册入口。因此生产站点只能使用预先创建的测试账号。

数据库中的 `profiles` 表引用 `auth.users`，但现有迁移没有在新 Auth 用户创建时自动插入 profile 的触发器。公开注册开放后，新用户即使完成邮箱验证，也可能缺少工作台所需的 profile。

### 2.2 窄屏控件重叠

工作台的日期控制使用固定三列布局，部分操作区域在较窄屏幕或较长中文文本下缺少换行、最小宽度和稳定的触控尺寸。异步按钮只替换文字，文字长度变化还可能导致布局跳动。

### 2.3 运行状态不明显

登录、保存和 AI 操作主要通过“登录中…”或“请求处理中”等文字表达。页面没有统一的动态图标，部分操作也没有 `aria-busy`，网络较慢时用户容易重复点击或误判网站失效。

## 3. 注册与邮箱验证

### 3.1 登录页模式

登录卡片提供“登录”和“注册”两个明确的模式切换按钮：

- 登录模式保留邮箱、密码和登录行为。
- 注册模式显示邮箱、密码、确认密码和创建账号按钮。
- 切换模式时清除旧错误和成功提示，但保留已输入的邮箱，减少重复输入。
- 密码至少 8 个字符；确认密码必须一致。
- 提交期间禁用模式切换和表单按钮，防止重复请求。

### 3.2 注册请求

浏览器调用 Supabase `auth.signUp`：

- `email` 和 `password` 使用用户输入。
- `emailRedirectTo` 指向当前站点的 `/auth/callback?next=/login?verified=1`。
- 不在前端展示 Supabase 原始错误详情。
- 对已存在邮箱也显示中性的查收邮件提示，避免泄露账号是否存在。

注册请求成功后不进入工作台，而是显示：

> 注册申请已提交，请打开验证邮件完成验证，然后返回本页登录。

### 3.3 验证回调

现有 `/auth/callback` 继续负责用 `code` 换取会话。邮箱验证完成后：

1. Supabase 跳转到生产站点回调地址；
2. 回调完成 code exchange；
3. 清除验证过程中创建的会话；
4. 跳转到 `/login?verified=1`；
5. 登录页显示“邮箱验证成功，请登录”。

这样满足“邮箱验证后再登录”，不会让用户点击邮件后直接进入工作台。

回调失败时仍返回固定的中文错误提示，不暴露认证服务细节。

### 3.4 Profile 自动创建

新增数据库迁移：

- 创建 `public.handle_new_user_profile()`，使用 `security definer` 和固定 `search_path`；
- 在 `auth.users` 插入后触发；
- 将新用户 ID 写入 `public.profiles`；
- `display_name` 优先读取安全的用户 metadata，否则使用空字符串；
- 使用冲突安全写入，避免重复回调造成错误；
- 回填已经存在但缺少 profile 的 Auth 用户。

RLS 继续限制用户只能读取和修改自己的 profile。

## 4. 响应式布局修复

### 4.1 工作台顶栏

- 标题区域允许收缩和换行。
- 退出按钮保持至少 44px 的触控高度。
- 退出错误放到独立行，不与按钮争夺横向空间。

### 4.2 日期控制

- 手机宽度下改为两行：日期输入独占主要宽度，上一天/下一天保持方形触控按钮，“回到今天”独占下一行。
- 不依赖固定的 `8.5rem` 最小列宽。
- 所有控件设置 `min-width: 0`，长日期本地化或浏览器原生图标不得撑破容器。

### 4.3 操作按钮与图标

- 学生卡片的编辑、删除按钮允许换行并保持间距。
- AI 三个操作按钮在窄屏下纵向排列，在空间足够时再展开。
- 按钮内容使用统一的内联布局，加载图标与文字有固定间距且不会覆盖。
- 关键按钮设置最小高度，文字换行时仍保持可点击区域。
- 页面继续禁止横向溢出，但不以裁切内容作为解决方案。

验收宽度至少覆盖 320px、375px、768px 和桌面视图。

## 5. 统一加载动画

新增轻量、无外部依赖的 `LoadingSpinner` 组件：

- 使用 CSS 边框旋转动画；
- 默认继承当前文字颜色；
- 装饰性图标使用 `aria-hidden="true"`；
- 支持小、中两个尺寸；
- 在系统开启 `prefers-reduced-motion: reduce` 时停止连续旋转并显示静态状态。

新增可复用的按钮内容组件或约定，使异步按钮统一呈现：

- spinner；
- 当前动作文字，例如“登录中”“注册中”“保存中”“生成中”“修改中”“归档中”“退出中”；
- `aria-busy="true"`；
- disabled 状态。

应用范围：

- 登录和注册；
- 新增/编辑学生；
- 删除学生；
- 每日记录自动保存状态；
- AI 生成、修改、归档；
- 退出登录；
- 工作台因日期或学生切换触发的页面导航。

自动保存仍保留“已保存”和失败重试文字；spinner 只在请求实际进行时显示。

## 6. 错误处理

- 注册字段错误在发请求前本地提示。
- 注册服务异常显示“注册暂时不可用，请稍后重试”。
- 登录仍区分凭据错误和服务不可用，但不显示供应商原始信息。
- 验证回调失败显示固定提示，并允许重新登录或注册。
- 所有失败场景恢复按钮可用状态，保留邮箱和非敏感输入。
- 密码不会写入日志、本地存储或 URL。

## 7. 测试策略

### 7.1 组件测试

- 登录页显示登录/注册切换。
- 注册模式显示确认密码。
- 密码过短和两次密码不一致时不调用 Supabase。
- 注册成功显示查收邮件提示且不导航到工作台。
- 注册异常显示固定中文错误并恢复按钮。
- pending 时按钮含 spinner、`aria-busy` 且不可重复提交。
- 学生表单、学生删除和 AI 操作的 pending 状态显示统一 spinner。

### 7.2 回调与迁移测试

- 验证回调成功后 sign out 并重定向到 `/login?verified=1`。
- 回调 exchange 或 sign out 失败时进入固定错误路径。
- SQL 迁移包含 Auth 用户 profile 触发器、回填和安全设置。

### 7.3 端到端与视觉验收

- 注册模式可访问并成功提交模拟注册。
- 已验证账号可以正常登录。
- 320px 和 375px 下日期控件、学生操作按钮、AI 按钮没有横向溢出或重叠。
- 桌面三栏布局保持不变。
- 所有关键异步操作在请求期间出现可见加载动画。

## 8. 部署要求

- 将新迁移应用到当前 Supabase 项目。
- Supabase Auth 保持邮箱确认开启。
- Supabase Site URL 设置为生产域名。
- Redirect URLs 包含生产域名的 `/auth/callback`，并可保留本地开发回调。
- 代码合并到 GitHub `main` 后由 Vercel 自动部署。
- 部署后用一个非测试邮箱完成注册、验证邮件、返回登录和创建学生的生产冒烟测试。

## 9. 不在本次范围

- 邀请码或管理员审批；
- 忘记密码和修改密码；
- Google、微信等第三方登录；
- 用户角色和团队共享；
- 自定义邮件模板；
- 全站骨架屏或复杂页面过渡动画。
